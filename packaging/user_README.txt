一览清图 — 使用说明（开箱即用包）
================================
批量清除图片水印，智能检测

启动
----
双击 RemovePicWatermark.exe
或双击 start.bat

本包已内置：Python 运行时、OpenCV、PyTorch、YOLO(ultralytics)、
LaMa 修补权重、YOLO 底座/可选检测权重、BiRefNet（样式 AI 抠图，若打包时已附带）。
无需再安装 Python。


一、推荐使用顺序
----------------
1. 菜单「设置」
   - 修补：高质量修补（推荐）/ 快速预览
   - 运行设备：自动 / CPU / GPU
     （高质量修补、训练检测、样式 AI 抠图共用此设置）

2. 水印样式
   - 导入一张带水印的样例图
   - 矩形框住一个水印单元（满屏重复字只框一个），或涂抹
   - 保存样式
   - 选中样式后可在页内：左改模板、右预览；重提取 / AI 抠图 / 保存

3. 训练检测（可选）
   - 导入多图框选水印 → 设轮数 → ▶ 训练
   - 完成后自动写入检测模型，批量「样式+模型 / 水印模型」可用

4. 批量去除
   - 添加图片或文件夹，勾选样式
   - 定位：
       固定位置 — 认定该处一定有，按录入位置与遮罩直接覆盖
       附近匹配 — 认定大致在该区域，用样式在附近比对后覆盖
       全图匹配 — 同款可能出现在任意位置，整图查找
   - 查找：
       水印样式 — 只用已保存样式匹配
       样式+模型 — 先样式；本张未检出/样式偏弱时才用检测模型补漏
       水印模型 — 主要用训练好的检测模型
   - 固定位置时查找锁定为「水印样式」（按区域实心矩形盖遮罩，不匹配）
   - 顶栏 ▶ 开始处理
   - 成品在「处理结果」中查看

5. 单张精修（难图）
   - 打开图 → 涂红 → ▶ 去除 → 导出才写盘


二、数据与模型目录（与 exe 同级）
--------------------------------
workspace/
  profiles/          水印样式
  jobs/              批量任务（input / masks / debug / output）
  models/
    torch/hub/checkpoints/big-lama.pt   LaMa 修补
    yolo/watermark.pt                   训练后的检测权重
    yolo/yolov8n.pt / yolov8n-obb.pt    训练底座
    birefnet/                           样式页 AI 抠图（BiRefNet）
  yolo_train/        训练数据与日志


三、显卡与 GPU 加速（用户必读）
------------------------------
本软件 GPU 加速依赖：NVIDIA 显卡 + 正确驱动 + 本包内置的 CUDA 版 PyTorch。

【能加速的功能】
  - LaMa 高质量修补
  - YOLO 训练
  - 样式页 BiRefNet AI 抠图
【始终用 CPU 的功能】
  - 模板匹配 / 残差扫描找水印
  - OpenCV 预览修补

【你需要准备什么】
  1. NVIDIA 独立显卡（AMD / Intel 核显不能用本包的 CUDA 加速）
  2. 安装较新的 NVIDIA 显卡驱动
     - 官网：https://www.nvidia.com/Download/index.aspx
     - 或笔记本厂商驱动中心
  3. 驱动装好后，命令行能运行：nvidia-smi
     - 能列出 GPU 名称与驱动版本即可
  4. 显存建议 ≥ 4GB（如 GTX 1650 4GB 可跑常规图；更大图更吃显存）

【关于 CUDA】
  - 用户一般不需要单独安装 CUDA Toolkit
  - 本包已内置带 CUDA 的 PyTorch；只要驱动够新即可
  - nvidia-smi 里显示的 “CUDA Version” 是驱动支持的上限，不是要你再装一套 CUDA

【在软件里怎么选】
  - 设置 → 运行设备 →「自动」：有 GPU 就用，没有则 CPU
  - 「GPU」：尽量用显卡；不可用时会回退 CPU
  - 「CPU」：强制 CPU（更慢，但兼容性最好）
  - 各页标题旁会显示当前设备摘要，例如：
    批量去除（自动 → GPU · NVIDIA …（4GB） · LaMa）

【没有显卡 / 驱动不行时】
  - 仍可正常使用，全部走 CPU
  - LaMa 与训练会明显变慢，可先用 OpenCV 预览检测是否正确


四、注意
--------
- 个人数据都在 workspace/，升级时可保留该文件夹
- 勿误删 workspace\models 下权重，否则离线能力会缺
- 精修多轮默认不自动存盘，导出才写入文件
- 训练占用磁盘与显存；小样本建议多框实例、轮数约 80–120
- 更完整说明见开发仓库 docs/（GPU加速与设备自检、去水印完整流程等）
